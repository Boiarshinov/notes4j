### Однострочные комментарии
`//` - текст за этим символом в строке не будет рассматривать компилятором
```java
int x = 5 //Присвоение переменной значения
```

### Многострочные комментарии
Многострочные комментарии записываются следующим образом
```java
/*
Многострочный
комментарий
*/
```

### Комментарии TODO
Такие комментарии собираются на отдельной вкладке IDE, чтобы ты потом мог вспомнить на чем ты остановился и что надо дописать.

```java
// todo дописать реализацию метода writeObject()
```

### Комментарии FIXME
Служат для того, чтобы показать программисту, что ему необходимо переработать (починить) какой-то участок кода. Собираются средой разработки на той же вкладке, что и TODO

```java
//fixme не пашет
```

### [Javadoc-комментарии](javadoc.md)
Служат для отображения подсказок при вызове методов
```java
/**
 * Моя любимая программа
 * @autor Бояршинов
 * @version 1.0
 */
```

### Рекомендации:
- Устаревший комментарий хуже отсутствующего
- Вместо комментария лучше отрефакторить код
